  • Options Bossy UI Tooltip currently supports:

    • directiveData choices:

      • text:
      • This property is just a raw string of text or can be a string of html markup. If it is a raw string it will be used as the tooltip text. If it is the markup, the markup will be displayed within the tooltip.
        • If you don’t want to pass the markup as a string for the “text:” property, you can place the HTML markup within the < bossy-tooltip > tags as long as the markup you are providing is wrapped within < div class=”tooltip-content” > tags. If one decides to provide HTML markup this way, you must set the “transclude” property in the directiveOptions to “True.”

    • directiveOptions choices:

      • align:
      • This property is used display the tooltip anchor either on the “left” or the “right”. if align is not set, the anchor is defaulted to the middle. Only works when position is set to default (top).
      • color:
      • This property changes the background color of the tooltip. It is defaulted to black. The possible colors are as follows: “green,” “orange,” “red,” and “blue.” This can be set dynamically after the tool tip is compiled. You can also pass in a color via a custom class.
      • position:
      • This property is used set the relative positioning of the tooltip. The default setting will display the tooltip above the anchor markup. This can also be set to “bottom”, “left”, or “right”.
      • type:
      • This property is used to define a more specialized tooltip such as “download” which sets up a download progress that can be adjusted by changing the value of options.progress.
      • transclude:
      • (the name of this option conflicts with the angular keyword, so the name is a placeholder.) Set this property to true to allow HTML markup to be placed within the < bossy-tooltip > tags.
      • persist:
      • This property forces the tooltip to display without the user hovering over the anchor text. This will allow the developer to show the tooltip programmatically. This can be set dynamically after the tool tip is compiled.
      • progress:
      • This property is used to communicate download progress if you are using the download tooltip. This can be set dynamically after the tool tip is compiled.
      • icon:
      • This property allows the developer to insert an ionicon into the tool tip. ionicon page
      • iconFloat:
      • This property allows the developer to float the icon left or right.
